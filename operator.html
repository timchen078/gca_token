Operator Boundaries

  • Local operator backend only; public self-service claiming remains off.
  • Read-only Base Mainnet eth_call / ERC-20 balanceOf wallet verification.
  • No private key, seed phrase, exchange API secret, withdrawal permission, custody request, signature, or transaction is required.
  • 100 credits and GCA Member records are local JSONL ledger records under .gca_access_data/.
  • The 10,000 GCA member benefit remains a manual reserve-wallet transfer review; this page never sends tokens.

Local API

GET /gca/operator-summary powers this console. GET /gca/review-package exports a local reviewer evidence package for support and platform follow-up. Individual records remain available through GET /gca/pre-registrations, GET /gca/wallet-verifications, GET /gca/credit-ledger, GET /gca/member-ledger, GET /gca/member-benefit-transfers, and GET /gca/member-review. POST /gca/member-benefit-transfers records a completed manual reserve-wallet transfer after the operator signs it outside this app and the backend verifies the receipt with read-only Base Mainnet data.
