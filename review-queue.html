GCA Review Queue Contract

This page defines the planned manual review workflow for GCA pre-registration, read-only wallet balance checks, 100 Web3 Radar utility credits, GCA Member status, and support review while public self-service claiming is not live.

The queue contract is not live today. It is not a public submission queue. It is a public operating contract for future controlled HTTPS account UI and internal operator review, not a place to submit private data or request custody.

Queue Lanes

Pre-Registration Intake

Records an account intent from the official member flow after approved HTTPS intake is connected.

Wallet Balance Review

Checks Base Mainnet chain ID 8453, the official GCA contract, and ERC-20 balanceOf.

Holder Credit Review

Marks 10,000 GCA eligibility for a planned one-time 100 Web3 Radar utility credits record.

GCA Member Review

Marks 1,000,000 GCA eligibility for planned GCA Member status after member ledger activation.

Support Case Review

Tracks support notes without overriding wallet verification or product release gates.

Platform Profile Follow-Up

Keeps BaseScan, GeckoTerminal, wallet-warning, and reviewer follow-up separate from user eligibility.

Operator Controls

Identity Checks

Use only Base Mainnet chain ID 8453 and contract 0x3197c42f4a06f7be32a9a742ac2a766f0ff682c6.

Read-Only Verification

Wallet review must use read-only eth_call and ERC-20 balanceOf; it must not request signatures or transactions for balance reading.

Audit Trail

Each queue action should record reviewId, registrationId, lane, status, updatedAt, reviewer note, and public evidence reference when the backend is live.

Do Not Collect

  • Private key or seed phrase.
  • Exchange API secret or withdrawal permission.
  • One-time code or recovery phrase.
  • Custody request, fund-transfer request, or live trading instruction.

Do Not Claim

  • Do not claim 100 utility credits are self-service claimable before controlled account UI and ledgers are live.
  • Do not claim GCA Member status is active before member ledger activation.
  • Manual support cannot override on-chain wallet-balance verification.
  • Do not claim exchange listing approval, completed third-party audit, mature liquidity, price support, or trading outcomes.
