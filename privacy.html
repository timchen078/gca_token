What The Public Page Collects

The current member pre-registration page creates a local pre-registration packet in the browser. The packet can include email, Telegram handle, Base wallet address, declared GCA balance, program intent, consent flags, generated registration ID, and timestamp. The public static site does not send this packet to a GCA server by itself.

directSubmissionEndpointConfigured is currently false on the public static page.

When Data Leaves The Browser

Data leaves the browser only when a user copies it, downloads it, emails it to GCAgochina@outlook.com, or later submits it through a controlled HTTPS account UI after that flow is published. Email submission is handled through the user's chosen email client and inbox provider.

Wallet Verification

Wallet verification is planned as a read-only ERC-20 balanceOf check against the GCA contract on Base Mainnet. GCA does not need private keys, seed phrases, exchange API secrets, withdrawal permission, or custody to verify a public wallet balance.

Future Controlled Intake

After controlled HTTPS account UI is live, GCA may store pre-registration fields, wallet verification results, credit ledger records, GCA Member status, review statuses, operational logs, and support notes needed to run the access bridge.

How Data May Be Used

Support review and user follow-upAllowed
Wallet balance eligibility checksAllowed
100 credits and GCA Member ledger recordsAllowed
Fraud, abuse, and duplicate-registration preventionAllowed
Private keys, seed phrases, or custody requestsNever

User Requests

Users can request correction or deletion by emailing GCAgochina@outlook.com. Some records may be retained when needed for abuse prevention, security review, operational integrity, or legal compliance. GCA does not sell pre-registration contact data.

Public Safety Boundary

No private key, seed phrase, exchange API secret, withdrawal permission, or custody request is part of the GCA member flow. Pre-registration data does not create a live Web3 Radar account, does not make credits claimable from this static page, and does not override wallet balance checks or risk controls.
