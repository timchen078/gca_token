用户路径

从注册到会员审核

1. 邮箱注册

用户先用 邮箱注册 加入 GCA 用户名单。这个步骤不需要钱包、不需要签名、不需要付款。

2. API 状态确认

公开接口状态、只读检查命令和管理员读取边界集中放在 中文 API 状态 页面。

3. 未来账户 UI

受控 HTTPS 账户 UI 上线后，才会进行正式钱包绑定、只读余额验证和账本激活。

4. 100 credits 审核

计划规则是每个注册用户绑定一个钱包，验证持有至少 10,000 GCA 后进入 100 Web3 Radar utility credits 审核。

5. GCA Member 审核

计划规则是购买并连续持有至少 1,000,000 GCA 满 30 天，才可以进入 GCA Member 状态审核。

6. 会员权益流程

通过 GCA Member 审核后，才进入一次性 10,000 GCA 会员权益审核；如通过，应来自项目或 owner-held reserve，不是新铸币。

安全边界

用户中心不会要求这些信息

  • 不会要求私钥、助记词、钱包密码、验证码、交易所 API Secret、提现权限或托管资金。
  • 钱包验证只能是只读余额检查，不需要签出资金，不需要授权转移资产。
  • 邮箱注册不会自动激活 100 credits、GCA Member 或 10,000 GCA 会员权益。
  • 会员权益不是现金、收入、报销、交易权限或风控豁免，也不是自动发放。
  • 任何后续交易、添加流动性或链上转账都必须由用户在自己的钱包里确认。
