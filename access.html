GCA Access Portal Blueprint

This page defines the planned controlled account path for GCA AI Quant Access: read-only wallet verification, 100 Web3 Radar utility credits, GCA Member ledger activation, 30-day holding-period review, 10,000 GCA member benefit review, and support review workflow.

The current state is blueprint only. The controlled HTTPS account UI is not live, direct submission is not connected, credits are not self-service claimable, GCA Member status is not self-service claimable, and the 10,000 GCA member benefit is not self-service claimable.

Planned User Journey

Step 1 Create account

Use a controlled HTTPS account UI after it is released. The static site does not create a live account today.

Step 2 Connect wallet

Use read-only wallet verification on Base Mainnet through eth_call and ERC-20 balanceOf.

Step 3 Review record

Create a support review record before any credit ledger activation or member ledger activation.

Step 4 Activate access

Eligible records can activate 100 credits, GCA Member status, or the one-time 10,000 GCA member benefit only after the required release gates are live.

Access Modules

Credits, member status, and the planned 10,000 GCA member benefit are account-level records only. They are not cash, income, reimbursement, trading permission, or transferable value, and they cannot bypass risk controls.

Required Controls

  • controlled HTTPS account UI
  • read-only GCA balance verification
  • credit ledger activation
  • member ledger activation
  • support review queue
  • privacy notice and participation terms
  • risk-control review and simulation or testnet first for any future trading workflow

Security Boundaries

  • No custody.
  • No withdrawal permission.
  • No exchange API secret collection.
  • No private key or seed phrase collection.
  • No live trading enabled.
  • No risk-control bypass.
