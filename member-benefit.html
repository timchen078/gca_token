GCA Member Benefit Review

This page defines the planned review path for the one-time 10,000 GCA member benefit. A user must buy and continuously hold at least 1,000,000 GCA for 30 consecutive days, then pass support and member-ledger review before any reserve-funded transfer can be approved.

1. Create Packet

The user submits official contact fields and one Base wallet address. No private key, seed phrase, exchange API secret, withdrawal permission, or custody request is allowed.

2. Check Balance

The review path checks the GCA token contract on Base Mainnet with a read-only ERC-20 balanceOf call.

3. Review 30 Days

The user must provide public transaction evidence that supports a 30-day continuous holding review for the threshold balance.

4. Ledger Decision

The member ledger can mark the case not_started, wallet_pending, holding_period_pending, needs_more_information, eligible, transferred, rejected, or closed.

5. Reserve Transfer

If approved, the one-time 10,000 GCA member benefit is sent from project or owner-held reserve. The token contract has no mint function.

6. Close Review

The review record should store the decision, transfer transaction hash if applicable, review date, and support notes.

Public Claim Boundaries

  • Safe to say: GCA has published planned GCA Member benefit review rules.
  • Safe to say: the planned rule requires buying and continuously holding at least 1,000,000 GCA for 30 consecutive days before benefit review.
  • Safe to say: the planned one-time member benefit amount is 10,000 GCA after support and ledger approval.
  • Do not say the 10,000 GCA member benefit is self-service claimable today.
  • Do not say holding 1,000,000 GCA automatically triggers a transfer without review.
  • Do not describe the member benefit as newly minted, cash, income, reimbursement, trading permission, voting control, lifetime access, or risk-control bypass.
