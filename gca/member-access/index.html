GCA Member Access

This controlled HTTPS account UI submits a GCA member account request, verifies a Base wallet with read-only ERC-20 balanceOf, and writes D1 ledger records for 100 Web3 Radar utility credits and GCA Member review.

The 10,000 GCA member benefit is not automatic. If a member ledger becomes active, the benefit remains pending manual reserve-wallet transfer review.

Access Boundaries

  • This UI does not request custody, approvals, signatures, private keys, seed phrases, exchange API secrets, or withdrawal permission.
  • Wallet verification is a backend read-only eth_call against the official GCA contract on Base Mainnet.
  • 100 credits and GCA Member status are utility ledger records, not cash, income, reimbursement, trading permission, or risk-control bypass.
  • The 10,000 GCA member benefit remains pending manual reserve-wallet transfer review after eligibility is recorded.

Platform-Only Evidence Path

The live API is hosted at https://gca-registration-api.gcagochina.workers.dev. Admin ledger reads are token-protected; public account submission and wallet verification do not expose the admin token.
