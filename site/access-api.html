GCA Access API Contract

This page defines the planned controlled HTTPS API contract for the GCA access bridge. It covers pre-registration, read-only wallet verification, credit ledger records, GCA Member records, and support review statuses.

The API contract is not live today. It is a public implementation contract for future backend work, not a public submission endpoint, account system, or self-service claim page.

A local operator backend is available at tools/gca_member_backend.py for localhost testing. It serves this static site, enables same-origin local submission, verifies GCA with read-only Base Mainnet eth_call, and writes local JSONL ledger records without making public self-service claiming live.

Endpoint Contract

POST /gca/pre-registrations

Receives account intent and gca_member_preregistration_v2 packet fields after controlled HTTPS account UI is live.

POST /gca/wallet-verifications

Runs read-only GCA balance checks using Base Mainnet eth_call and ERC-20 balanceOf.

GET /gca/credit-ledger

Returns account-level 100 Web3 Radar utility credits records after ledger activation.

GET /gca/member-ledger

Returns GCA Member ledger state, holding-period review, 10,000 GCA member benefit status, next refresh due date, and status after activation.

POST /gca/support-review

Creates or updates support review records for manual review workflows.

GET /gca/member-review

Reads review status for the authenticated account after the controlled account UI exists.

Required Controls

  • controlled HTTPS origin
  • authenticated account session
  • CSRF protection for state-changing routes
  • rate limits on pre-registration and wallet verification
  • structured audit logs for status changes
  • server-side validation of chain ID 8453 and the GCA contract address

Do Not Collect

  • Private key or seed phrase.
  • Exchange API secret.
  • Withdrawal permission.
  • Custody request or fund-transfer request.
  • One-time code or recovery phrase.
  • Any permission that bypasses risk controls.
