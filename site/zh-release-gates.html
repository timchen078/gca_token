Go-Live Gates

当前上线状态和剩余门槛

受控 HTTPS 账户 UI

账户资料提交已经接入 Workers + D1。后续仍要补充更完整的登录态、用户状态页和支持处理后台。

只读 GCA 余额验证

钱包验证只能读取 Base Mainnet 上 GCA 的 ERC-20 balanceOf，不需要私钥、助记词、签名或授权转移资产。

100 credits 账本

10,000 GCA 持有者的 100 Web3 Radar utility credits 必须进入可审计账本后，才能说可以使用。

GCA Member 账本

1,000,000 GCA 连续持有 30 天的审核结果必须写入会员账本，不能只凭截图或口头确认。

支持审核队列

会员权益、资料补充、异常情况和人工复核必须进入支持审核队列，不能跳过审核直接处理。

风控和模拟环境

未来任何量化交易相关功能，必须先经过风控检查、模拟盘或测试环境，不能直接连接真实交易执行。

Security Boundary

上线门槛里的安全底线

  • 不索要私钥、助记词、钱包密码、验证码、交易所 API Secret 或提现权限。
  • 不托管用户资金，不让用户把资产转入所谓客服、审计、保证或解锁地址。
  • 不允许绕过风控直接进行真实交易执行。
  • 不把 credits、会员身份或会员权益描述成现金、收入、报销、交易权限或风控豁免。
  • 不把产品计划、JSON 文件或本地测试说成正式公开产品功能；账户入口和账本记录可以按当前事实说明为已上线。

Release Sequence

推荐推进顺序

1. 邮箱注册继续使用 register.html 和 Cloudflare Workers + D1 收集用户邮箱。
2. 运营同步按 中文运营流程 同步 D1 注册记录、应用退订名单并导出 public-redacted 联系 CSV。
3. 账户 UI继续完善受控 HTTPS 用户中心，用于查看注册状态、提交公开链上资料和查看审核结果。
4. 钱包验证接入只读 GCA 余额验证，不发起交易、不要求签名授权转移资产。
5. 账本激活上线 100 credits 账本和 GCA Member 账本，并保留可审计记录。
6. 支持审核上线支持审核队列，处理 30 天持有证明、资料补充和会员权益审核。
7. 量化产品先发布模拟盘、回测、风险提醒和复盘工具，再评估任何真实执行功能。
