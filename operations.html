GCA Access Operations Runbook

This runbook defines how GCA operators should handle manual access requests before public self-service account UI, public submission queue, credit ledger writes, and GCA Member ledger writes are live.

The runbook is public documentation only and is not live today. It is not a live backend, not a public submission queue, not a public support ticket system, not an account system, and not a way to request custody, withdrawals, live trading, or balance-verification overrides.

Manual Operator Workflow

1. Intake Triage

Accept only non-sensitive account intent, public wallet address, declared GCA balance, program intent, and public transaction references.

2. Identity Check

Confirm Base Mainnet chain ID 8453 and official contract 0x3197c42f4a06f7be32a9a742ac2a766f0ff682c6.

3. Wallet Balance Check

Use read-only eth_call and ERC-20 balanceOf. Do not request signatures or transactions for balance reads.

4. Eligibility Decision

Mark 10,000 GCA holder credit eligibility and 1,000,000 GCA Member eligibility without writing live ledger records yet.

5. Support Reply

Reply with status, next step, and public links. Do not promise approval timing, platform listing, price support, audit approval, or trading results.

6. Ledger Handoff

Only after controlled account UI and ledgers are live, create credit or member records through the reviewed backend path.

7. Platform Follow-Up

When a reviewer asks for more information, send public URLs only and keep pool, contract, status, and claim boundaries consistent.

8. Closure

Close the record only after status, evidence reference, reviewer note, and next public link are recorded without secrets.

Operational Controls

No Sensitive Data

Reject Private key or seed phrase, Exchange API secret or withdrawal permission, one-time code, recovery phrase, Custody request, fund-transfer request, or live trading instruction.

No Manual Override

Manual support cannot override on-chain wallet-balance verification, release gates, credit ledger activation, or member ledger activation.

Public Boundary

Every response should keep GCA described as concept-stage access planning until controlled HTTPS account UI and ledgers are live.

Safe Reply Template

Your request has been received for manual review. GCA access claiming is not self-service yet. We can review your public Base wallet address with a read-only GCA balance check, but we will never ask for private keys, seed phrases, exchange API secrets, withdrawal permission, custody, or fund transfers.

Unsafe Reply Boundary

Do not state that credits are live, GCA Member status is active, listings are approved, liquidity is deep, a third-party audit is complete, wallet warnings are permanently removed, or any trading outcome is expected.
